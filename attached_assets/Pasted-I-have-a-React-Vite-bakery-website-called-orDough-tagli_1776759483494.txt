## 2. MONGODB INTEGRATION

Replace all localStorage usage in the Admin Dashboard (src/App.jsx) with MongoDB 
via a Node.js/Express backend.

Backend setup (create server/ folder):
- server/index.js — Express server on port 3001 with CORS enabled
- server/db.js — Mongoose connection using MONGODB_URI from .env
- server/routes/ — separate route files:
  - customers.js  (GET all, POST new, PUT update, DELETE)
  - orders.js     (GET all, POST new, PUT update status, DELETE)
  - menu.js       (GET all, POST, PUT, DELETE)
  - ingredients.js
  - recipes.js
  - reviews.js
  - settings.js

Mongoose schemas:

Customer schema:
{
  name: String (required),
  phone: String,
  email: String,
  address: String,
  totalOrders: Number (default 0),
  totalSpent: Number (default 0),
  createdAt: Date (default now),
  notes: String
}

Order schema:
{
  customerId: ObjectId ref Customer,
  customerName: String,
  date: Date,
  items: [{ menuItemId: String, name: String, emoji: String, qty: Number, price: Number }],
  discount: Number,
  deliveryCharge: Number,
  status: String enum [pending, ready, delivered, cancelled],
  notes: String,
  invoiceNumber: String (auto-generated: ORD-YYYYMMDD-XXXX),
  createdAt: Date
}

In the frontend:
- Replace all useLocalStorage hooks with React Query (useQuery / useMutation) 
  hitting the Express API at http://localhost:3001/api/
- Show loading spinners during fetch
- Toast notifications on success/error (use react-hot-toast)
- Add a new "Customers" tab in the admin sidebar with a customer list, 
  search by name, view order history per customer
